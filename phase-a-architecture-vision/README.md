# Phase A — Architecture Vision

The motivation layer for one ADM iteration: who cares about Forge
during this iteration, why, and what target state the architecture
exists to reach. Forge-wide *meta*-stuff (single architect, the
capability-trajectory method, containers-only, single-server)
lives one layer above in
[`../phase-preliminary/`](../phase-preliminary/) and is not
re-stated here.

Product portfolio (which products and their value streams) lives
in Phase B, not here.

## Layout

- [`vision.md`](vision.md) — vision statement (this iteration).
- [`stakeholders.md`](stakeholders.md) — architect of record,
  future operator, end users.
- [`drivers.md`](drivers.md) — what time we are trying to save.
- [`goals.md`](goals.md) — TTS / PTS / EB / Architect-velocity.
- [`principles.md`](principles.md) — *iteration-scoped* principles
  only (none today). The forge-wide meta-principles live in
  [`../phase-preliminary/architecture-principles.md`](../phase-preliminary/architecture-principles.md).
