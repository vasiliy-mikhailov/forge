# Phase A — Architecture Vision

The motivation layer: who cares about Forge, why, and what target
state the architecture exists to reach. Product portfolio (which
products and their value streams) lives in Phase B, not here.

## Layout

- [`vision.md`](vision.md) — vision statement.
- [`stakeholders.md`](stakeholders.md) — architect of record, future
  operator, end users.
- [`drivers.md`](drivers.md) — what time we are trying to save.
- [`goals.md`](goals.md) — TTS, PTS, EB, Architect-velocity (the four
  Motivation-layer goals; Phase H trajectories quantify them).
- [`principles.md`](principles.md) — single architect, capability
  trajectories, containers-only, single-server.
- [`adr/`](adr/) — Phase A scoped ADRs.

The forge-level [`AGENTS.md`](../AGENTS.md) keeps a one-paragraph
synthesis of this phase plus pointers into this folder; the depth
lives here.
