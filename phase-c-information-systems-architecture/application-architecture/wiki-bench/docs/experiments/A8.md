## Hypothesis (IF–THEN–BECAUSE)

> **IF** поднять vLLM `max_model_len` с 65536 до 131072 (YaRN factor
> 2.0 → 4.0) и перезапустить compiler-лабу, **THEN** тот же агентский
> прогон (Qwen3.6-27B-FP8 + skill v1 + source 000) пройдёт до commit'а,
> **BECAUSE** изначальный T3 crash (`Unterminated string at char 285`)
> при F1-доказанной модельной справности — это симптом усечения output'а
> по budget'у контекстного окна: T3 prompt ~50K input + body ~16K output
> = ~66K, что превышает 65K окно.

## Methodology

T3 single-source full pipeline. Конфигурация:
- `forge/.env`: `INFERENCE_MAX_MODEL_LEN=131072` (было 65536).
- `phase-c-information-systems-architecture/application-architecture/wiki-compiler/docker-compose.yml`:
  `--hf-overrides.rope_scaling.factor 4.0` (было 2.0).
- `phase-c-information-systems-architecture/application-architecture/wiki-bench/configs/models.yml`: для qwen3.6-27b-fp8
  `max_model_len: 131072`.
- остальное по умолчанию: kv-cache fp8, qwen3_xml parser, skill v1 без
  правок.

Compose `make kurpatov-wiki-compiler-down/up` → vLLM healthy, GPU 92/96 GB.
Запуск: `./run.sh` (НЕ `run-battery.sh` — он перепатчивает .env).

### Quality observations

- Source 000 на 37% короче Opus baseline. Возможные причины:
  (a) модель меньше capability/coherence чем Opus, (b) агент не следует
  инструкции «≥ 200 строк» строго. Проверить через D1 (skill v2:
  явное минимальное число bullet points в каждой секции).
- 6 концептов vs Opus 32 — модель не разворачивает каждый понятийный
  элемент. Это quality, не bug.
- 3 «security_risk» errors — известное поведение OpenHands SDK: модель
  должна выставлять это поле в каждом tool call (LOW/MEDIUM/HIGH/UNKNOWN);
  Qwen иногда забывает. Recovery работает, но шумит лог. Possible fix:
  patch skill add explicit instruction. Не критично.

### Что это меняет в стратегии

- **The Big Bet ушла третий раз** (см. spec.md §1.3). Теперь корень
  понятен: KV-budget. Не модель, не skill, не парсер.
- **D-cluster (skill rewrite)** более не нужен для решения crash'а.
  Может быть нужен для quality — но это другая задача.
- **B-cluster (другие модели)** становится top priority — теперь
  имеет смысл прогонять всю батарею с 128K, чтобы выбрать best
  cheap candidate.
