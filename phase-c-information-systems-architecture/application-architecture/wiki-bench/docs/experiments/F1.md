# F1 — microbench `L*` (JSON tool-args length threshold)

Закрытый детальный спек. Методология лаборатории — в
[`../spec.md`](../spec.md). Список гипотез — в
[`../backlog.md`](../backlog.md). Status: **done** (REFUTED).

## Hypothesis (IF–THEN–BECAUSE)

> **IF** запустить серию `chat/completions` к vLLM с tools-spec
> `file_editor` и требованием создать .md файл с Cyrillic body длины
> *L* ∈ {1, 2, 4, 8, 12, 16, 24, 32, 40, 49} KB, по 2–10 trials каждой длины,
> **THEN** наблюдаемая pass-rate (валидный JSON args, прошедший
> schema-validator) монотонно убывает с ростом *L*, и можно определить
> пороговое значение **L\* = max L при pass-rate ≥ 0.95**,
> **BECAUSE** наблюдаемый T3 краш Qwen3.6-27B-FP8 (`Unterminated string
> at char 285`) детерминированный, длино-зависимый и не зависит от
> reasoning'а — модель просто рвёт строку посреди args.

ICE = 8 × 9 × 9 = **648** (top of backlog). Это instrumentation-ставка:
без `L*` все остальные гипотезы (A1–A5, D1–D3, B-cluster) проверяются
дорого и качественно. С `L*` они проверяются на T1 за минуты и
количественно.

## Methodology

T1 microbench, без OpenHands-сэндбокса. Прямой call в vLLM
`/v1/chat/completions` с tools-spec `file_editor` (минимальная схема,
mirror OpenHands SDK).

Промпт:

> Создай файл /tmp/test.md с заданным содержимым, используя инструмент
> file_editor (command="create"). Содержимое:\n\n{BODY}

`BODY` генерируется детерминированно: сначала `# Заголовок документа`
+ повторяющийся блок `## Раздел №{n}\n\n[Cyrillic параграф]\n\n`, режется
по байтам UTF-8 до точно `L` байт (выравнивание границы кодпойнта).

Артефакт скрипта генерации: [`run_microbench.py`](../../evals/microbench/run_microbench.py).

## Falsifiability criteria (заявлены до запуска)

F1 опровергнута, если **хотя бы одно**:

1. между двумя прогонами одной длины *L* pass-rate отличается > 30 п.п.
   ⇒ результат недетерминирован, понятие «порога» неприменимо;
2. pass-rate **не** монотонно убывает с ростом *L*
   (например при L=8 KB прошло 10/10, а при L=4 KB — 4/10);
3. pass-rate(L=1 KB) < 0.95 ⇒ модель падает безотносительно длины,
   причина не в размере → нужна другая гипотеза.

## Parameters

### Fixed

- model: `Qwen/Qwen3.6-27B-FP8`
- vLLM:
  ```
  --tool-call-parser qwen3_xml
  --reasoning-parser qwen3
  --max-model-len 65536
  --kv-cache-dtype fp8
  --quantization fp8
  --default-chat-template-kwargs '{"enable_thinking": false}'
  ```
- `max_completion_tokens`: **32768** (в payload)
- `tool_choice`: `auto`
- temperature: 0
- prompt template: статический (см. `run_microbench.py:PROMPT`)
- генератор body: фиксированный seed per-L, sha256-стабильный

### Variables

| variable     | levels                                                  |
| ------------ | ------------------------------------------------------- |
| L (байты)    | {1, 2, 4, 8, 12, 16, 24, 32, **40**, 49} × 1024         |
| trial_index  | [0, 9] (для L ∈ {1, 2, 4}); [0, 1] (для L ≥ 8 KB)        |

(Изменено vs spec: для L ≥ 8K сократили trials до 2 после наблюдения
полной детерминированности первых трёх cells при temperature=0.

Total: 10×3 + 2×7 = 44 одиночных completion-вызова.

**`L*_isolated` (Qwen3.6-27B-FP8, qwen3_xml, max_tok=32768, temp=0) ≥ 49 152 bytes.**

Дополнительные наблюдения:

- `args_len ≈ L_target + 600 байт` overhead (ключи `command`, `path`,
  плюс JSON-квоутинг Cyrillic символов).
- Latency масштабируется линейно по L: ~4.4 ms/byte ≈ устойчивые 47 t/s
  output throughput на Blackwell. Соответствует логам vLLM (`Avg
  generation throughput: 47.4 tokens/s`).
- Полная детерминированность: внутри cell все trials идентичны
  (`body_hash` совпадает, `args_len` идентичен, latency колеблется
  ≤ 0.05 с). При temperature=0 + одинаковом prompt'е модель выдаёт
  байт-в-байт одинаковый output.

### Конкретные следствия для backlog

- **A1 (max_completion_tokens cap)**: сильно ↑ Confidence (с 2 → 5). Это
  теперь главная подозреваемая. ICE c 120 → 250.
- **D1/D2/D3 (skill rewrite)**: Confidence ↓ (была 7-8 → 3-4). Если
  модель не имеет архитектурного длиного-bug'а, переписывать skill — не
  обязательно. ICE D2 c 256 → 96 (всё ещё в backlog как страховка, но
  не приоритет).
- **H6 (THE BET — skill v2)**: пересмотр. The Bet больше не «skill
  написан под frontier». Новый THE BET — A1 (max_tokens cap).
- **H2 (Qwen3.6 family bug)**: ICE ↓ (Confidence c 2 → 1). Если bug не
  размеро-зависимый, и Qwen3.6 на этом тесте проходит, то и Qwen3.5/3.0
  тоже скорее всего пройдут — но это уже неинтересный ablation.
- **H8 (context starvation)**: ICE ↑. Новая формулировка: «при ~50K
  входного контекста + max_tokens=4K (по умолчанию OpenHands), модель
  не успевает выдать 16K-токенный body».
- **H10 (новая)**: создаём — «T3 crash вызван исключительно
  max_completion_tokens cap'ом OpenHands». Falsifier: при `max_tokens=32768`
  тот же T3 прогон проходит до commit'а.

### Risk to North Star

**Снижен.** Если A1 верна — фикс одной строкой в `bench/run.sh` (либо
OpenHands LLMConfig override). Не нужно ни переписывать skill, ни
менять модель, ни строить guided JSON decoding.
