### F1. ADR 0015 dec 5 closed: per-runner JSONL score logs live

Predicate: meta. Paths:

- `scripts/test-runners/_score_history.py` — NEW shared module:
  - `append_scores(runner_name, results)` — writes one JSONL row
    per case per run.
  - `latest_per_case(path)` / `previous_per_case(path)` —
    readers for P21.
  - `regressions(path, score_drop_threshold=0.10)` — returns
    `[(test_id, prev_row, curr_row), …]` for cases whose latest
    is worse (verdict ladder regression OR score drop ≥ 10%).
- `scripts/test-runners/.score-history/` — new directory:
  - `test-auditor-runner.jsonl` (12 rows; baseline)
  - `test-wiki-pm-runner.jsonl`  (17 rows; baseline)
  - `test-lab-AGENTS-runner.jsonl` (16 rows; baseline)

  Combined: **45 baseline rows** committed. All verdicts PASS.

**Schema** (one JSON object per line):
```
{
  "ts":         "2026-05-01T18:40:23Z",
  "git_commit": "feff3d606c",
  "runner":     "test-auditor-runner",
  "test_id":    "AU-05",
  "verdict":    "PASS",
  "score":      5.0,
  "score_max":  6.0,
  "threshold":  3.0,
  "detail":     "score=5.0/6.0; C1=1, C2=1, …"
}
```

`score`, `score_max`, `threshold` are nullable for cases without
a Reward function (today: AU-02, AU-04, AU-04b, etc. — these are
binary inspection tests where the verdict alone carries the
signal).

**CLI ergonomics.** All 3 runners now accept `--log-scores`:

- Default behaviour unchanged: prints results, exit code reflects
  any FAIL.
- `--log-scores`: appends one row per case to the runner's JSONL
  *in addition* to printing.
- Pattern selection (`AU-1*`, `wiki-bench`, `LA-WC-*`) still
  works alongside `--log-scores`.

**Discipline.** Interactive dev runs (sanity-checking after an
edit) MUST NOT pass `--log-scores` — that would inflate history
with intermediate noise. Log only "milestone" runs: before/after
a material edit to a role md or runner; before each audit walk
(so P21 has fresh data); on commits that touch test-suite or
runner code.

### F2. P21 introduced — score-history regression detection

Predicate: meta. Path:
`phase-h-architecture-change-management/audit-process.md`.

**Symptom.** P21 added below P20. Walks the JSONL files; for each
test_id with ≥ 2 rows, flags:
- *verdict regression*: PASS → italian-strike, PASS → FAIL,
  italian-strike → FAIL → `WARN`,
- *score drop ≥ 10% with verdict held* → `INFO`.

First-run cases (only 1 row in JSONL) cannot regress and are
silently skipped.

**This walk's P21 result:** 45 cases logged, 0 with ≥2 runs,
0 regressions. Baseline state. Future walks will compare against
this commit's rows.
