### F1. K2-R2 ran against real lecture A — L1 gate FAILED honestly

Predicate: meta. Path:
[`/phase-f-migration-planning/experiments/K2-compact-restore.md`](../phase-f-migration-planning/experiments/K2-compact-restore.md).

**Symptom.** All 4 L1 variants on real lecture A
(9963 words, `kurpatov-wiki-raw/data/Психолог-консультант/000
Путеводитель/000 Знакомство…/raw.json`) score trip-quality
0.0000 → 0.0116 (V4 winner). The K2 spec's L1 hypothesis gate
(≥ 0.20) FALSIFIES. Forward recall is 0.909 (10/11) — capped by
one missing Substance observation (OBS-A-001's Selye-attributed
verbatim is not a literal quote from raw A).

**Note.** RLVR did its job — the synth-fixture-tuned algorithm
that PASSed the gate at trip-quality 0.2261 on synth dropped
20× on real data. Three insights captured in the K2 Post-Mortem
section:

1. *Synth-vs-real gap.* Whisper acts as an implicit L0 Air-strip
   (0 vocalised hesitations survive transcription); real
   Курпатов uses **discourse markers** ("значит" 30×, "вот"
   101×, "то есть" 32×, "на самом деле" 17×, "собственно" 19×,
   "как бы" 10×, "ну, " 9×) — patterns the V1..V4 set doesn't
   target. **V5_discourse_markers queued.**
2. *Corpus-observations data-quality issue.* OBS-A-001's
   verbatim isn't grep-able in raw A (0 hits for "Селье",
   "Ганс", "опираться"). Wiki PM follow-up: re-walk source A
   with `--literal-only` discipline. Without the fix, K2-R2
   forward recall is permanently capped at 10/11.
3. *Whisper artefacts.* Real `raw.json` schema is
   `{info, segments[]}`, not `{transcript, …}`. `compact.py`
   gained `_normalise_raw()` to handle both. Future ingest
   regressions land here.

### F4. K2 sweep containerised per P3 (Dockerfile + compose)

Predicate: P3. Paths:
- `phase-c-…/wiki-bench/compact_restore/Dockerfile.k2-sweep` —
  slim Python 3.12, no host-pip, no GPU, no network at runtime.
- `phase-c-…/wiki-bench/compact_restore/compose.k2-sweep.yml` —
  bind-mounts forge tree (read-only) + kurpatov-wiki-raw vault
  (read-only) + tmpfs `/tmp` (64 MB); resource caps (256 MB,
  1.0 CPU); runs as non-root user.

**Symptom.** The earlier `k2_r2_README.md` activation procedure
called bare `python3 sweep.py` on the host — a P3 violation if
run on `mikhailov.tech`. Updated activation steps now use
`docker compose run --rm k2-sweep` exclusively for the
production path; bare-metal `python3` is documented as the
**dev-sandbox-only** invocation (the architect's local Cowork
host).

**Note.** "Don't make my server a mess" is the architect's
P3-compliance ask. The Cowork sandbox where I ran today's
K2-R2 measurement IS bare-metal (allowed for dev), but the
Blackwell production path uses the container.
