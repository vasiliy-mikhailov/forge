### F1. source-author + concept-curator reclassified from Application-Component scaffolding to Phase B Business Roles

Predicate: P14 + ArchiMate metamodel discipline. Paths:
- `phase-b-business-architecture/roles/source-author.md` — NEW.
- `phase-b-business-architecture/roles/concept-curator.md` — NEW.

**Symptom.** Pre-audit, these two operational agent personas
existed only as informal references in
`phase-c-information-systems-architecture/application-architecture/wiki-bench/AGENTS.md`
prose (e.g. "L2: real concept-curator OpenHands sub-agent
integration") and as stubs in `tests/synthetic/test_source_coordinator_*.py`.
Per ArchiMate (and TOGAF Phase B/C separation): the
**wiki-bench harness** is an Application Component; the
**LLM agents** that run inside it filling specific behaviours
(compile one source; curate one concept) are Business Actors
filling Business Roles. Conflating Actor + Component into the
Application Component file misses the Role layer entirely —
and without a Role md, there's no motivation chain, no
decision-rights, no escalation path, no test-md / reward
function attribution.

**Note.** Both new role md files follow the established Wiki
PM / Auditor / Developer / DevOps template:

- **Source-author** — compiles ONE raw transcript into ONE
  source.md per activation. Activates from the wiki-bench
  SKILL.md inside the OpenHands SDK harness with wiki-compiler
  vLLM as the model backend. Decision rights: TL;DR phrasing,
  section ordering, NEW vs REPEATED markers,
  `concepts_touched` slug picks (within the existing graph).
  Escalates: schema violations → Wiki PM; voice-preservation
  regression → Wiki PM + Architect; new-concept needed →
  hand-off to concept-curator.
- **Concept-curator** — maintains `data/concepts/` graph (one
  slug operation per activation, create OR update). Spawned
  as a sub-Conversation from source-author when
  `concepts_introduced` is non-empty. Decision rights: slug
  naming, definition phrasing within the SKILL budget,
  Related-concept link picks, when to mark a concept stale.
  Escalates: genuine concept-graph contradictions → Wiki PM
  + Architect; slug-rename across the whole graph → Wiki PM
  (then Developer ships find-replace); new schema fields →
  Wiki PM.

Both roles realise the **Service operation** capability of
[`forge-level.md`](../phase-b-business-architecture/capabilities/forge-level.md)
(the "agents do operations" axis the architect named) plus
specific **Voice preservation / Reading speed /
Concept-graph quality / Dedup correctness** dimensions of
[`develop-wiki-product-line.md`](../phase-b-business-architecture/capabilities/develop-wiki-product-line.md).

Tests today: PENDING per-role test md (SA-NN / CC-NN cases +
reward functions queued). Transitive coverage by:
- `phase-c-…/wiki-bench/tests/synthetic/test_source_coordinator_*.py`
  (unit + integration + e2e — exercises the source-author →
  concept-curator hand-off with a stub concept-curator).
- `phase-c-…/wiki-bench/tests/synthetic/test_verify_source_*.py`
  (pass-condition contract).
