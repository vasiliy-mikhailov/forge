## Findings — verdict WARN

### F1. wiki-pm.md missing `## Motivation chain` section

Predicate: P14. Path:
`phase-b-business-architecture/roles/wiki-pm.md`.

**Symptom.** The role file does not contain a `## Motivation
chain` section. ADR 0015 decision point 1 requires every role
file to declare the chain (Driver → Goal → Outcome → Capability →
Function → Role) it serves. The `auditor.md` role added in the
prior commit (`75926c6`) has the section; `wiki-pm.md` predates
ADR 0015 and was not amended.

**Rule.** [ADR 0015](../phase-preliminary/adr/0015-verifiable-agent-rewards.md)
decision point 1.

**Proposed fix or escalation.** Add a `## Motivation chain`
section to `wiki-pm.md` mirroring the shape used in `auditor.md`:

- *Driver*: lecture-time-saving (per `phase-a/drivers.md`).
- *Goal*: Architect-velocity + TTS (Phase A).
- *Outcome*: every implementation choice cites a requirement;
  no orphan rules accumulate.
- *Capability*: Develop wiki product line / Requirement
  traceability dimension.
- *Function*: `wiki-requirements-collection.md` walk.
- *Role*: Wiki PM.

### F2. Catalog row R-D-orchestration-kvcache lacks Goal/Driver source

Predicate: P15. Path:
`phase-requirements-management/catalog.md`.

**Symptom.** The Source cell reads `"Phase D analysis"`. That is
the *origin phase* of the requirement, not a Goal or Driver from
the Motivation chain. P15 expects Source cells to cite either a
Phase A Goal (TTS, PTS, EB, Architect-velocity), a Driver from
`phase-a/drivers.md`, or an upstream Phase F closure (e.g.
"G3 close-out"). All other Phase D rows comply (G2/G3 close-out
references); only this one is the outlier.

**Rule.** ArchiMate Motivation chain (Driver → Goal →
Requirement) per
[`../phase-preliminary/archimate-language.md`](../phase-preliminary/archimate-language.md)
§ Motivation Domain.

**Proposed fix or escalation.** Replace `"Phase D analysis"`
with the actual upstream that surfaced the requirement —
likely `"G2 closeout: KV-cache hit rate measured at 0%"` or
similar concrete trace.

### F6. Two of three Drivers do not explicitly cite a Goal

Predicate: P18. Path:
`phase-a-architecture-vision/drivers.md`.

**Symptom.** The drivers.md file has 3 bullets:

- *"Time spent consuming information from Russian psychology
  lectures…"* — no Goal name in the bullet text.
- *"Time spent writing/optimizing programs in domains where RL
  with verifiable rewards (RLVR) can do the slog automatically."*
  — no Goal name.
- *"Architect-velocity: every minute the architect spends
  recovering GPUs…"* — does cite the Architect-velocity Goal
  (by name).

The Driver → influences → Goal relationship is implicit-prose
in the first two; readable but not typed.

**Rule.** ArchiMate Motivation domain § 6.3.2 (Driver) +
relationship verb *Influence* (§ 5.2.3) per
[`../phase-preliminary/archimate-language.md`](../phase-preliminary/archimate-language.md).

**Proposed fix or escalation.** Append explicit Goal references
to each Driver bullet. Suggested:

- "…Russian psychology lectures …  → *influences* TTS."
- "…RLVR can do the slog automatically. → *influences* TTS,
  PTS."
- (Architect-velocity already names the Goal; add the verb
  "→ *influences* Architect-velocity" for consistency.)

### F7. Goals PTS and EB have only self-referential rows in the catalog

Predicate: P19. Paths:
- `phase-requirements-management/catalog.md` rows R-A-PTS,
  R-A-EB.

**Symptom.** Both Goals (PTS = Practical Time Saved; EB =
Economic Balance) appear in the Phase A section of the catalog
as their *own* placeholder rows, with explicit blocker notes
("Blocked on user count > 1"; "Implicit; no explicit measurement
today"). No Phase B or Phase D trajectory cites them, because
they're deferred. P19 nominally requires every Goal to have a
realising trajectory; the placeholder rows acknowledge the
deferral with rationale, which is consistent with the architecture
method.

**Note.** This is the documented-deferral case the P19 verdict
note explicitly exempts ("a Goal may legitimately be deferred —
see R-A-PTS / R-A-EB rows that explicitly note their blockers;
those are exempt"). Reported as INFO so the deferral stays
visible — when one of these unblocks, P19 will start producing
real findings if no trajectory follows.
