### F14. Multiple commit-author identities on `main`

Predicate: P1. Path: forge git history.

**Symptom.** `git log --format='%an' main | sort -u` returns six
identities:
- `Author <author@example.com>` (clear configuration leak)
- `Cowork Session` (agent-attribution)
- `Vasiliy Mikhailov` (architect, full name)
- `forge bench` (agent-attribution)
- `vasiliy-mikhailov` (architect, GitHub handle)
- `vmihaylov` (architect, short handle)

**Rule.** [Architecture principle 1](../phase-preliminary/architecture-principles.md):
single architect.

**Note.** Three of the six are the architect under different name
strings — not a violation, but suggests git config inconsistency
across machines. `Cowork Session` and `forge bench` are agent
attribution and acceptable. `Author <author@example.com>` is the
likely-bad row — that's a default-template identity, not the
architect.

**Proposed fix.** Audit prior commits; rewrite history to
unify the architect's name (or accept the cosmetic inconsistency
since git history is immutable). Address the `Author
<author@example.com>` identity by setting `user.name` /
`user.email` correctly on every machine the architect commits
from.
