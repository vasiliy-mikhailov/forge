### F6. Two-pass sweep discipline learned (postmortems-worthy)

Predicate: meta. (Not added to postmortems.md in this commit; queued.)

**Symptom.** Same one-pass-with-overlapping-source-target trap from
audit-2026-05-01z F3 nearly recurred when I started writing the goal-cascade
backfill — almost did `s/Service operation/Quality (Service-operation)/g`
across all phases without checking each chain's intent first. Caught
myself by listing the 11 P29 fails first and writing per-file replacements,
not a regex sweep.

**Note.** The discipline that emerged: when a class of fails crosses
multiple intent-categories, hand-write the per-file fix. Regex sweeps are
appropriate when the source-target mapping is 1:1 (e.g. ADR 0022's heading
rename); they're a trap when intent varies per file (e.g. cascade-target
varies by which Capability the file realises).
