### F1. ADR 0019 — every motivation chain Outcome cites a measurement source

Predicate: P10 + P14. Path:
[`phase-preliminary/adr/0019-measurable-outcomes.md`](../phase-preliminary/adr/0019-measurable-outcomes.md).

**Symptom.** Pre-this-commit
`scripts/test-runners/motivation-measurability-report.py`
surfaced **34 of 116 artifacts have prose Outcomes with no
cited measurement source** — chains exist (P24 PASS) but a
reader cannot pull a current Level for the current commit
without re-running heuristics inside the report. Same root
cause as pre-P7: enforcement was piecemeal (Roles via test
runners; tests via Reward functions; R-NN via catalog Status),
no first principle universalised it. ADR 0019 closes the
loop: every chain's `**Outcome**:` bullet MUST be followed by
a `**Measurement source**:` line citing one of 10 standard
formats (runner / runner-aggregate / lab-tests / audit-predicate /
catalog-row / experiment-closure / corpus-walk / customer-cycle /
n/a — declarative / n/a — pending).

**Note.** ADR 0019 itself complies (citation:
`audit-predicate: P26`).

### F4. motivation-measurability-report.py parser updated

has two `## Motivation` sections — rationale + self-applied
chain — and the parser was picking the wrong one).
