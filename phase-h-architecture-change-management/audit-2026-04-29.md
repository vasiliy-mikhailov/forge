# Forge docs audit — 2026-04-29

Re-audit driven by the question: после серии TOGAF-restructures за день
(Preliminary phase + Requirements Management + per-phase per-entity
splits + lab-level template propagation + voice-preservation product
requirement), what is misaligned right now?

## Findings — fixed in this re-audit

### F1. ADR 0007 was at the wrong phase

`phase-b-business-architecture/adr/0007-labs-restructure-self-contained-caddy.md`
contained two interlocked decisions: (i) a repo-organisation rule
("each lab is a self-contained directory"), and (ii) a technology-
component placement rule ("per-lab caddy fronting :80/:443; MLflow
inside rl-2048"). Neither is Business Architecture content. Per
[`framework-tailoring.md`](../phase-preliminary/framework-tailoring.md)
the ADR's primary phase is Phase G (governance / repo organisation),
with cross-link from Phase D for the caddy and MLflow placement.

**Fix:** moved to `phase-g-implementation-governance/adr/`. Path
references in `README.md`, `tests/README.md`, `tests/smoke.md`, and
`wiki-bench/SPEC.md` updated. Phase B `adr/` folder is now empty;
no Phase B ADRs exist today (no business-architecture decisions
have warranted one since the product portfolio was defined).

### F2. Long-tail of `${STORAGE_ROOT}/<old>` and `forge/<old>/` paths

`rl-2048/mlflow/SPEC.md` still pointed at `${STORAGE_ROOT}/mlflow/mlruns`
(pre-ADR-0007 layout). `wiki-bench/SPEC.md` had `forge/rl-2048/` as a
top-level path. `phase-g/operations.md` had `kurpatov-wiki/sources/`
near a disaster-recovery sentence. `wiki-ingest/SPEC.md` referenced
`kurpatov-wiki/Dockerfile.pusher` (the file moved into the lab).

**Fix:** all four updated to current paths. Historical ADRs that
document state-at-the-time keep their original references.

### F3. `wiki-compiler/SPEC.md` internal inconsistency

Line 89 said `Default INFERENCE_MAX_MODEL_LEN=65536`; line 127 said
the default is `131072`. Same file, two values, neither is a
"current vs target" pair — just a stale paragraph that was missed
in the YaRN-factor-4.0 / 128K context update.

**Fix:** line 89 rewritten to match reality (131072).

### F4. `README.md` recovery checklist references non-existent target

Disaster-recovery step said `make setup && make base`. `make base`
does not exist (per the 2026-04-25 audit, never has). Replaced with
`make setup` then bring up the lab(s) you need.

## Findings — fixed (closed by architect request 2026-04-29)

### N1. ADR 0001 (monorepo-flat-structure) → Preliminary ✓

The decision "single git repo, no submodules" is fundamentally an
architecture-repository setup choice. Moved from
`phase-g-implementation-governance/adr/` to
`phase-preliminary/adr/`. Preliminary previously had no ADRs; it
now anchors the meta-architecture decisions distinct from Phase G's
operational governance.

### N2. ADR 0009 (ssh ControlMaster) → Preliminary ✓

ssh-ControlMaster is workstation-side tooling for the architect's
edit loop. Reframed as an architecture-capability decision (how the
architect operates the platform) rather than a forge technology
component. Moved from `phase-d-technology-architecture/adr/` to
`phase-preliminary/adr/`. The G3 experiment cross-link to this ADR
was updated.

### N3. Per-lab historical-ADR path references ✓

Added a "Path map for current readers" amendment block at the top
of each affected historical ADR:

- `wiki-ingest/docs/adr/0005-split-transcribe-and-push.md`
- `wiki-ingest/docs/adr/0007-wiki-layer-mac-side.md`
- `wiki-ingest/docs/adr/0008-ingest-dispatch.md`
- `wiki-compiler/docs/adr/0001-vllm-public-openai-compatible-endpoint.md`

The body text — which uses `kurpatov-wiki/` and `inference/` paths
that were correct at the time — stays intact. The new amendment
block maps each old path to its current equivalent, so a present-day
reader can find the actual files without reading the historical
record as if it were current.

This is the pattern for any future historical ADR that survives a
path rename: don't rewrite the body, add a top-of-file path map.

### N4. Bench's three top-level docs ✓

Trimmed `wiki-bench/README.md` (was 95 lines, now 50) to a thin
casual-reader entry point: identity statement + paragraph on what
the lab does + pointers to `AGENTS.md` and `SPEC.md`. The substantive
content (full operator runbook + spec) stays in `SPEC.md` and
`AGENTS.md`. Each of the three docs now has a distinct audience:

- `README.md` — casual GitHub reader; what is this lab.
- `SPEC.md` — engineer / operator; the technical contract.
- `AGENTS.md` — agent / future-architect; Phase A-H scoped context.

Other labs may or may not warrant a `README.md` of their own; bench
is the busiest of the four and benefits from a separate entry-point
file.

## Conclusion

After today's restructure + this re-audit + the N1-N4 closure pass, the architecture is
internally consistent at the level "every doc is in the right
phase, no current-state doc carries a stale path, no two phases
disagree on the same fact, no current-state doc carries a status
flag the trajectory model rejects." The historical-ADR exemption
remains a useful escape hatch from delete-on-promotion: ADRs are
durable records of decisions, and the path/name references inside
them are part of the historical record.

The 2026-04-25 audit doc is now functionally closed; per the
trajectory model, it goes to git history on the next sweep. This
2026-04-29 audit replaces it as the most recent working-tree audit.
