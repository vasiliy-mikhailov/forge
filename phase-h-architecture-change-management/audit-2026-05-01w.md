## Findings — verdict INFO

### F1. ADR 0018 — privacy boundary public/private split

Predicate: P10 + P14. Path:
[`phase-preliminary/adr/0018-privacy-boundary-public-vs-private-repos.md`](../phase-preliminary/adr/0018-privacy-boundary-public-vs-private-repos.md).

**Symptom.** Pre-this-commit forge contained 30 Cyrillic
verbatim quotes from Kurpatov lectures in
`phase-b-business-architecture/products/kurpatov-wiki/corpus-observations.md`
— IP leakage to the public `vasiliy-mikhailov/forge` GitHub
repo. ADR 0018 codifies the boundary: forge holds the
**architecture metamodel** (schema, structure, predicates);
private repos hold the **content** (verbatims, paraphrases,
ledger entries). Mirrors the standard "open-source the
framework, keep the data private" pattern.

**Note.** ADR 0018 also confirms `lecture_A_synth.json` +
`synth-corpus-observations.md` + `bloated-fixture.md` stay in
forge (synthetic content the architect explicitly authored).
Marker convention: `synth` / `_template` / `_fixture` in name
OR under `/tests/synthetic/fixtures/`.

### F2. corpus-observations.md split: schema-only stub in forge; full content in private wiki repo

Predicate: P14 + P11. Paths:
- `phase-b-business-architecture/products/kurpatov-wiki/corpus-observations.md`
  (forge, public) — replaced 468-line file with 99-line
  schema-only stub.
- `kurpatov-wiki-wiki/metadata/corpus-observations.md`
  (private) — full content with 30 Cyrillic verbatims +
  per-OBS classifications.

**Symptom.** The split preserves the methodology (S1+S2
process, observation schema, OBS-NNN ID convention,
source-letter mapping, count totals, bucket/dimension catalog)
in the public reference for any architect adopting the same
Wiki PM cycle for a different corpus. The content (verbatim
quotes, per-OBS classifications) lives next to the curated
source.md / concept.md outputs in the private repo.

**Note.** Tools that need verbatim access
(`measure-corpus-recall.py`) updated to default to the private
path via the new `_find_private_corpus_obs()` helper —
mirrors the `_find_repo` helper for sibling-repo discovery.
