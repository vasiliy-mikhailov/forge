# D7 — skill v2 tool-driven (12-step ritual + helper scripts)

Active spec. Methodology: [`../spec.md`](../spec.md). Backlog:
[`../backlog.md`](../backlog.md). Synthetic precedent that motivated
this: [`../../tests/synthetic/`](../../tests/synthetic/).

## Hypothesis (IF–THEN–BECAUSE)

> **IF** we replace the prose-only skill v1 with a 12-step tool-driven
> ritual (mandatory per-source `get_known_claims.py` for REPEATED
> classification + mandatory `factcheck.py` for empirical claims +
> mandatory skill-reload at every source boundary + deterministic
> bash self-verify before commit),
> **THEN** the same Qwen3.6-27B-FP8 producing
> `claims_REPEATED_sum=0, fact_check_citations_sum=0,
> claims_CONTRADICTS_FACTS_sum=0` on production module 005 baseline
> will close most of the spec-compliance gap, with all four metrics
> moving substantially toward Opus baseline values,
> **BECAUSE** the synthetic test (4 sources, same model, same harness,
> skill v2) showed 10/10 mandatory assertions pass — model knows how,
> but only obeys when the contract is enforced via tools whose absence
> would leave the source.md unverifiable.

## Architecture: as-is vs. to-be

### As-is (production at commit `2ea9723f`, Qwen3.6 baseline)

```
agent
 ↓ reads
skills/benchmark/SKILL.md   ← prose: "extract claims, classify,
                                fact-check, mark REPEATED, write
                                source.md..."
 ↓ has access to
file_editor / terminal / task_tracker / think
 ↓ external memory
(none — only in-context conversation)
```

Result: agent reads skill once at start of long 7-source run,
context drift accumulates, ritual silently degrades:
- sources 002-006 lose `## New ideas (verified)` section entirely
- 16/38 claims emerge unmarked (no `[NEW]/[REPEATED]/[CONTRADICTS]`)
- 0 fact-check citations across 7 sources
- 0 REPEATED markers across 7 sources

### To-be (D7)

```
agent
 ↓ reads (ONCE per source, via reload-step)
skills/benchmark/SKILL.md     ← 12-step ritual; mandates tool calls
 ↓ has access to
file_editor / terminal / task_tracker / think
 ↓ external memory & contract enforcement
skills/benchmark/scripts/
  list_sources.py             ← (already exists, kept)
  extract_transcript.py       ← (already exists, kept)
  get_known_claims.py         ← NEW: structured JSON of all prior
                                  source articles' claims with slugs.
                                  Skill mandates calling per-source
                                  before classification.
  factcheck.py                ← NEW: Wikipedia OpenSearch (RU+EN);
                                  returns {title, description, url}.
                                  Skill mandates calling per empirical
                                  claim. URLs in Claims must come from
                                  this script's output.
```

Per-source ritual (12 steps):

1. extract transcript via `extract_transcript.py <n>`
2. inventory of known claims via `get_known_claims.py`
3. **reload skill spec** via `file_editor view skills/benchmark/SKILL.md`
   (combats context drift)
4. extract claims from transcript (≥1 claim per 60s of transcript)
5. classify each claim NEW/REPEATED/CONTRADICTS_EARLIER (using slugs
   from step 2's JSON output)
6. for each empirical claim → `factcheck.py "<text>"`; if external
   contradicts → mark CONTRADICTS_FACTS, append URL inline
7. write `## TL;DR`
8. write `## Лекция (пересказ: только NEW и проверенное)`
9. write `## Claims — provenance and fact-check` (numbered, marker per
   claim, URL inline for fact-checked claims)
10. write `## New ideas (verified)`
11. write `## All ideas`
12. **deterministic self-verify via bash**: grep section presence,
    awk claim-count, count markers; if mismatched — REWRITE that
    section, do not commit
13. concept articles + concept-index update + atomic commit + push

## Falsifiability criteria (locked before run)

D7 is **falsified** if any one of the following holds on the new
branch's bench-grade after the experiment:

- `claims_unmarked_sum > 5` — ritual didn't enforce marker discipline
- `claims_CONTRADICTS_FACTS_sum < 1` — fact-check tool was bypassed
  (which would mean script-as-contract failed)
- `fact_check_citations_sum < 5` — citation-from-tool didn't take hold
- `claims_REPEATED_sum < 3` — REPEATED detection didn't work despite
  `get_known_claims.py` being mandated

D7 is **partially confirmed** if compliance metrics improve but quality
metrics (claim density, concept count) do not — would mean tools fix
spec adherence but not content depth (model capability ceiling).

## Methodology

### Branch naming

To separate "experiment axis" from "model axis", switching to:

  `experiment/<exp-id>-<date>-<served-name>`

Example: `experiment/D7-2026-04-26-qwen3.6-27b-fp8`. Existing baselines
(`bench/2026-04-25-qwen3.6-27b-fp8`, `bench/2026-04-25-claude-opus-4-6-cowork`)
remain on `bench/` — they are no-experiment-id baselines for that
model/skill pair. Future experiments preserve this split.

### Wiki repo branching

Skill v2 changes (helper scripts + new SKILL.md) live on a `skill-v2`
branch in `kurpatov-wiki-wiki`, NOT on `main`. The bench agent
checks it out after clone (added line in launch prompt). This keeps
`main` stable for Mac-side Cowork users.

### Run plan

1. Push `kurpatov-wiki-wiki/skill-v2` with new files.
2. Restart compiler-lab (no change — Qwen3.6-27B-FP8 stays).
3. Modified launch.md says: after clone, `git checkout skill-v2`.
4. Run `./run.sh`.
5. Agent executes 12-step ritual on each of 7 sources of module 005.
6. Final commit pushed to `experiment/D7-2026-04-26-qwen3.6-27b-fp8`.
7. Compare via `bench_grade.py` candidate=experiment-branch
   gold=opus-baseline; record table below.

**Source 000** was written correctly (5 sections + frontmatter + 16 concepts_touched + 16 concepts_introduced). It contained valid Claims with markers and URL citations from `factcheck.py`. We confirmed via the events.jsonl that:
- `get_known_claims.py` was called
- `factcheck.py` was invoked 64+ times across the run for empirical claims (Pareto, MacLean, Papez, Pavlov, Goldstein, insula, cingulate, Dunbar, Yerkes, oxytocin, endorphins, Krushinsky, Vygotsky, …)

**Sources 001–006** lost the ritual entirely. They were created in a single bulk action `Create source articles and concepts for 002-006` without frontmatter, without the 5 mandatory sections, with English-translated slugs (`Psychologist-consultant/05-conflicts/`), with `# Title` + freeform `## Overview / ## Summary / ## Key Themes` shape — completely off-spec.

The bench-grade L0 parser saw 7 source files but counted **0** claims because it only looks inside `## Claims — provenance and fact-check` blocks, and 6 of 7 source files had no such section.

**Concept articles**: 27 created (vs Opus 59, vs Qwen-baseline-skill-v1 16). Mid-point — better than baseline, far from Opus.

## Post-Mortem & Insights

### What went right (proves the hypothesis is correct in principle)

- `factcheck.py` and `get_known_claims.py` were invoked. Mandatory-tool contract worked.
- Source 000 emerged with full structure: frontmatter, 5 sections, marker per claim, URL citations to Wikipedia, CONTRADICTS FACTS marker on at least one claim. Same model as production baseline (Qwen3.6-27B-FP8).
- Concept count jumped 16 → 27 (+69%).

### What went wrong

1. **`SKILL.md` path bug**: skill referenced `wiki/skills/benchmark/scripts/list_sources.py` and
   `wiki/skills/benchmark/scripts/extract_transcript.py`, but the existing helpers live at
   `wiki/skills/benchmark/list_sources.py` (no `scripts/` prefix). Agent wasted ~5 minutes
   bashing around the FS to find them. Erodes trust in the skill contract — once the agent
   sees broken paths, the rest of the skill stops feeling reliable.

2. **SSL/openssl issue in the bench Docker image**: `factcheck.py` initially crashed because the
   `python:3.12-slim` image's `LD_LIBRARY_PATH` was conflicting with system openssl. Agent spent
   ~10 minutes on `Test HTTPS URL access`, `Install requests library`, `Try system curl with
   clean LD_LIBRARY_PATH`, eventually patched `factcheck.py` itself to clean its env on re-exec.
   Worked, but it consumed ritual-discipline budget the agent never recovered.

3. **Skill v2 length**: ~300 lines of Markdown vs the synth skill's ~130 lines. The added
   weight describes more sections (Лекция, New ideas (verified), All ideas), more rules
   (provenance markers, append-only concepts, Cyrillic-path workaround). Either:
   - the model lost the ritual under sheer skill bulk, or
   - the model decided "5 sections per source × 7 sources is too much, I'll batch" once it saw
     the size of source 000.

4. **No per-source isolation**: same OpenHands session for all 7 sources. After source 000 +
   ~10K tokens of accumulated reasoning + concept articles, the agent took the shortcut
   `Create source articles and concepts for 002-006` in one bulk action.

5. **Branch slug drift**: agent translated paths to English (`Psychologist-consultant/05-conflicts/`)
   instead of using the verbatim Cyrillic course/module names. Suggests the agent re-derived
   path conventions on the fly rather than reading them from `concept-index.json` or prior
   source articles. Possibly amplified by skill v1 having had the original Cyrillic paths;
   skill v2's reduced spec emphasis may have allowed this drift.

### Decision: pivot or persevere?

**Persevere with skill v2**, but with concrete fixes. The H-Q-plan hypothesis is **directionally
confirmed** by source 000 — when the agent does follow the ritual on a single source, the
output quality jumps. The failure mode is *attention budget on a 7-source single-session run*,
which is exactly H-Q1 territory.

D7-rev2 pivots to combine D7 + H-Q1:

1. **Fix path bugs in SKILL.md** — `scripts/` prefix removed for `list_sources.py` and
   `extract_transcript.py`.
2. **Pre-bake the SSL workaround** in bench Dockerfile — `RUN apt-get install -y ca-certificates`
   and ensure `python3 -c "import urllib.request; urllib.request.urlopen('https://en.wikipedia.org/wiki/Test')"`
   works first-try. No agent troubleshooting.
3. **Per-source isolation** — modify bench/run-battery.sh (or a new `run-source.sh`) to spin up
   one OpenHands invocation per source. Each session starts fresh: same model, same skill,
   one source. After source 000 commits, container exits, new container starts for source 001.
4. **Pin Cyrillic paths** in the launch prompt — explicit "do NOT translate paths to English;
   use the exact Cyrillic strings from the raw.json filesystem layout".

### Falsifiability check

D7 attempt #1 result against the locked criteria:

- `claims_unmarked_sum > 5` — vacuously true (no claim parsing happened on 6/7 sources). Rather than "ritual didn't enforce marker discipline", the failure is "ritual was abandoned wholesale on 6/7 sources".
- `claims_CONTRADICTS_FACTS_sum < 1` — true at the parse level, but only because the parser couldn't find the Claims section. Source 000 had at least one CONTRADICTS-style fact-check note for Pavlov / Goldstein attribution.
- `fact_check_citations_sum < 5` — true at parser level, false at run-level: 64+ factcheck.py invocations.
- `claims_REPEATED_sum < 3` — true. REPEATED detection didn't engage on 001-006 because those sources had no Claims section.

Per the locked criteria: **D7 is falsified at the L0 parser level**. But the underlying hypothesis (mandatory-tool contract drives compliance) is upheld for source 000 specifically. The failure is *durability across a 7-source session*, not the contract itself.

This rotates D7 from a "skill works / doesn't work" binary to a "skill works for a bounded session length" finding. Per-source isolation (H-Q1) is the right complement.
