# F1 — microbench `L*` (JSON tool-args length threshold)

Активный детальный спек. Методология лаборатории — в
[`../spec.md`](../spec.md). Список гипотез — в
[`../backlog.md`](../backlog.md).

## Hypothesis (IF–THEN–BECAUSE)

> **IF** запустить серию `chat/completions` к vLLM с tools-spec
> `file_editor` и требованием создать .md файл с Cyrillic body длины
> *L* ∈ {1, 2, 4, 8, 12, 16, 24, 32, 48} KB, по 10 trials каждой длины,
> **THEN** наблюдаемая pass-rate (валидный JSON args, прошедший
> schema-validator) монотонно убывает с ростом *L*, и можно определить
> пороговое значение **L\* = max L при pass-rate ≥ 0.95**,
> **BECAUSE** наблюдаемый T3 краш Qwen3.6-27B-FP8 (`Unterminated string
> at char 285`) детерминированный, длино-зависимый и не зависит от
> reasoning'а — модель просто рвёт строку посреди args.

ICE = 8 × 9 × 9 = **648** (top of backlog). Это instrumentation-ставка:
без `L*` все остальные гипотезы (A1–A5, D1–D3, B-cluster) проверяются
дорого и качественно. С `L*` они проверяются на T1 за минуты и
количественно.

## Methodology

T1 microbench, без OpenHands-сэндбокса. Прямой call в vLLM
`/v1/chat/completions` с tools-spec `file_editor` (схема берётся из
OpenHands SDK 1.17.0 — она у нас в `bench/openhands`).

Промпт:

> Создай файл `/tmp/test.md` с заданным содержимым, используя инструмент
> `file_editor`. Содержимое:\n\n{BODY}

`BODY` генерируется детерминированно per-(L, seed). Python-скрипт
строит повторяющийся валидный markdown с Cyrillic заголовками и
абзацами, режет/паддит до точно L байт UTF-8 (±32 байта tolerance).

Артефакт скрипта генерации: `evals/microbench/scripts/gen_body.py`.
Он же фиксируется в git, чтобы прогон был воспроизводим.

## Falsifiability criteria

F1 опровергнута, если **хотя бы одно** из:

1. между двумя прогонами одной длины *L* pass-rate отличается > 30 п.п.
   ⇒ результат недетерминирован, понятие «порога» неприменимо;
2. pass-rate **не** монотонно убывает с ростом *L*
   (например при L=8 KB прошло 10/10, а при L=4 KB — 4/10);
3. pass-rate(L=1 KB) < 0.95 ⇒ модель падает безотносительно длины,
   причина не в размере → нужна другая гипотеза.

Любой иной исход — новые данные, не подтверждение F1. Решение
pivot/scale принимается в Post-Mortem.

## Parameters

### Fixed

- model: `Qwen/Qwen3.6-27B-FP8`
- vLLM:
  ```
  --tool-call-parser qwen3_xml
  --reasoning-parser qwen3
  --max-model-len 65536
  --kv-cache-dtype fp8
  --quantization fp8
  --default-chat-template-kwargs '{"enable_thinking": false}'
  ```
- max_completion_tokens: дефолт vLLM (фиксируется в логе run'а)
- temperature: 0
- prompt template: статический (`evals/microbench/prompts/F1.md`)
- генератор body: фиксированный seed per-L, sha256-стабильный

### Variables

| variable     | levels                                       |
| ------------ | -------------------------------------------- |
| L (байты)    | {1, 2, 4, 8, 12, 16, 24, 32, 48} × 1024      |
| trial_index  | [0, 9]                                       |

Total: 9 × 10 = 90 одиночных completion-вызовов, ~1–3 сек каждый.
Wall ~5–15 мин включая warmup.

### DoE matrix shape

Полно-факторно (9 × 10 ячеек). Variance внутри (L)-cell даёт
σ(pass-rate). Тренд по L даёт `L*`.

## Evaluation

### Primary metric

**`L*`** = max L при pass-rate ≥ 0.95 (≥ 9.5/10 trials с валидным
JSON, прошедшим schema-validator для `file_editor`).

Fall-back **`L*_50`** = max L при pass-rate ≥ 0.5 (когда `L*` не
существует — pass-rate всегда < 0.95).

Запись в `evals/microbench/L_star.csv`:

```
date, model, parser, max_tok, L_bytes, trials, passed, pass_rate
```

### Guardrail metrics (delta к spec.md §5)

Все стандартные guardrails применяются. Дополнительно для F1:

- **Tool-args length**: фактическая длина `tool_call.arguments` в bytes
  логируется per-trial. Если она **меньше** заданного L — модель
  усекла body на стороне промптинга (тоже информативно — в Post-Mortem).
- **stop_reason**: фиксируется per-trial. Если массово `length` ⇒
  гипотеза A1 (max_tok cap) внезапно становится приоритетнее.

### Test set

T1 only. Если F1 не фальсифицирована, это **не означает**, что мы
переходим на T2/T3 — `L*` интересен сам по себе как метрика для
последующих гипотез.
